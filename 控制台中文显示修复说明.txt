======================================
  控制台中文显示修复说明
======================================

修复日期：2025-10-11

【问题描述】
在 Windows CMD 控制台中运行 start-admin.bat 时，Node.js 输出的中文日志显示为乱码。

【根本原因】
1. Windows 中文系统的 CMD 默认使用 GBK 编码（代码页 936）
2. Node.js 默认输出 UTF-8 编码
3. 两者编码不匹配导致中文乱码

【解决方案】
修改 Node.js 的 console.log 输出，将字符串从 UTF-8 转换为 GBK 编码后再输出到控制台。

【修改的文件】

1. main.js
   - 移除了设置 chcp 65001 的代码
   - 重写 console.log 和 console.error 函数
   - 使用 iconv-lite 将输出内容转换为 GBK 编码
   - 如果转换失败，自动降级为原始输出方式

2. start-admin.bat
   - 移除了 chcp 65001 设置
   - 保持 CMD 默认的 GBK 编码（代码页 936）

【技术细节】
使用 iconv.encode(message, 'gbk') 将 UTF-8 字符串转换为 GBK 编码的 Buffer，
然后通过 process.stdout.write(buffer) 输出到控制台。

【测试方法】
1. 关闭当前运行的应用
2. 运行：.\start-admin.bat
3. 观察控制台输出，中文应该正常显示，例如：
   - "执行命令: netsh interface show interface"
   - "=== 开始禁用接口: waiwang"
   - "IPC收到禁用请求: waiwang"

【备注】
- PowerShell 对 UTF-8 支持更好，如果使用 PowerShell，可以运行 start-admin.ps1
- 开发者工具中的控制台始终正确显示 UTF-8 编码的中文
- 只有 Windows CMD 需要特殊处理

【兼容性】
- Windows 7/8/10/11 中文版 ✓
- PowerShell 5.x/7.x ✓
- Node.js 12.x 及以上 ✓
