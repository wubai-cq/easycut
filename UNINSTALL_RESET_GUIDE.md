# EasyCut 卸载时重置激活码功能

## 功能说明

在应用卸载时自动调用重置激活码脚本，清除用户的激活信息，确保激活码的安全性。

## 实现方案

### 方案1：NSIS安装脚本（推荐）
- **文件**: `installer.nsh`
- **配置**: `package.json` 中的 `nsis.include`
- **优点**: 集成到安装包中，自动执行
- **适用**: 使用 electron-builder 打包的安装程序

### 方案2：PowerShell脚本（备用）
- **文件**: `uninstall.ps1`
- **优点**: 独立脚本，可以手动执行
- **适用**: 手动卸载或特殊场景

### 方案3：Node.js脚本（开发测试）
- **文件**: `test-reset.js`
- **优点**: 开发阶段测试使用
- **适用**: 开发和测试阶段

## 文件结构

```
c:\code\wifi\
├── installer.nsh          # NSIS安装脚本
├── uninstall.ps1          # PowerShell卸载脚本
├── uninstall-reset.js      # Node.js卸载脚本
├── test-reset.js           # 重置测试脚本
├── test-uninstall.js       # 卸载测试脚本
└── UNINSTALL_RESET_GUIDE.md # 本说明文档
```

## 使用方法

### 1. 开发阶段测试
```bash
# 测试重置功能
node test-reset.js

# 测试卸载重置
node test-uninstall.js
```

### 2. 打包时集成
```bash
# 打包安装程序（会自动包含NSIS脚本）
npm run build:installer
```

### 3. 手动执行重置
```bash
# 使用Node.js脚本
node test-reset.js

# 使用PowerShell脚本
powershell -ExecutionPolicy Bypass -File uninstall.ps1
```

## 功能效果

### ✅ 卸载时自动执行
1. **检测激活文件**: 检查 `C:\Users\[用户名]\.easycut\activation.json`
2. **删除激活文件**: 如果存在则删除
3. **清理目录**: 如果 `.easycut` 目录为空则删除
4. **显示进度**: 在卸载过程中显示重置进度

### ✅ 安全保护
1. **防止激活码泄露**: 卸载时清除激活信息
2. **保护激活码安全**: 防止他人使用已激活的激活码
3. **保持唯一性**: 确保激活码的唯一性

### ✅ 用户体验
1. **重新安装需要激活**: 重新安装后需要重新输入激活码
2. **激活码可重用**: 同一个激活码可以重新使用
3. **透明操作**: 用户可以看到重置过程

## 测试验证

### 测试步骤
1. **激活应用**: 使用测试激活码激活应用
2. **模拟卸载**: 运行 `node test-uninstall.js`
3. **验证重置**: 检查激活状态是否已重置
4. **重新安装**: 重新启动应用，应该显示激活界面

### 预期结果
- ✅ 激活状态重置为未激活
- ✅ 激活文件被删除
- ✅ 重新安装后需要重新激活
- ✅ 激活码可以重新使用

## 注意事项

1. **权限要求**: 需要用户目录的读写权限
2. **错误处理**: 如果删除失败，会显示错误信息但不影响卸载
3. **兼容性**: 支持Windows、macOS、Linux系统
4. **备份**: 建议在重要操作前备份激活文件

## 故障排除

### 问题1：重置失败
**原因**: 权限不足或文件被占用
**解决**: 以管理员身份运行或手动删除激活文件

### 问题2：NSIS脚本不执行
**原因**: 安装包配置问题
**解决**: 检查 `package.json` 中的 `nsis.include` 配置

### 问题3：PowerShell脚本被阻止
**原因**: 执行策略限制
**解决**: 使用 `-ExecutionPolicy Bypass` 参数
