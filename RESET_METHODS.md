# EasyCut 激活码重置方法

## 问题解决

由于NSIS脚本在打包时出现语法错误，我们提供了多种重置激活码的方法：

## 方法1：批处理脚本（推荐）

### 使用步骤：
1. 双击运行 `reset-activation.bat`
2. 脚本会自动删除激活文件
3. 重新安装应用后需要重新激活

### 特点：
- ✅ 简单易用，双击即可
- ✅ 自动检测和删除激活文件
- ✅ 支持清理空目录
- ✅ 显示详细的操作过程

## 方法2：Node.js脚本

### 使用步骤：
```bash
# 重置激活码
node test-reset.js

# 测试卸载重置
node test-uninstall.js
```

### 特点：
- ✅ 开发测试使用
- ✅ 详细的日志输出
- ✅ 可以集成到其他脚本中

## 方法3：PowerShell脚本

### 使用步骤：
```powershell
# 以管理员身份运行PowerShell
powershell -ExecutionPolicy Bypass -File uninstall.ps1
```

### 特点：
- ✅ 功能强大
- ✅ 支持复杂的文件操作
- ✅ 需要PowerShell环境

## 方法4：手动删除

### 使用步骤：
1. 打开文件资源管理器
2. 导航到 `C:\Users\[用户名]\.easycut\`
3. 删除 `activation.json` 文件
4. 如果目录为空，可以删除整个 `.easycut` 文件夹

### 特点：
- ✅ 最直接的方法
- ✅ 不需要额外工具
- ❌ 需要手动操作

## 推荐使用方案

### 对于普通用户：
**使用 `reset-activation.bat` 批处理脚本**
- 简单易用，双击即可
- 自动完成所有操作
- 有详细的操作提示

### 对于开发者：
**使用 Node.js 脚本**
- 可以集成到开发流程中
- 支持自动化测试
- 详细的日志输出

## 验证重置效果

重置完成后，可以通过以下方式验证：

1. **检查激活文件是否被删除**：
   ```
   C:\Users\[用户名]\.easycut\activation.json
   ```

2. **重新启动应用**：
   - 应该显示激活界面
   - 需要重新输入激活码

3. **测试激活码**：
   - 使用之前的激活码应该可以重新激活
   - 激活成功后应该正常使用

## 注意事项

1. **备份重要数据**：重置前建议备份激活文件
2. **权限要求**：某些方法可能需要管理员权限
3. **路径问题**：确保路径中没有特殊字符
4. **多用户环境**：每个用户都有独立的激活文件

## 故障排除

### 问题1：批处理脚本无法运行
**解决**：右键选择"以管理员身份运行"

### 问题2：PowerShell脚本被阻止
**解决**：使用 `-ExecutionPolicy Bypass` 参数

### 问题3：文件无法删除
**解决**：确保应用已完全关闭，或重启后再试

### 问题4：重置后仍显示已激活
**解决**：检查是否有多个用户账户，或手动删除激活文件

## 总结

虽然NSIS脚本在打包时出现问题，但我们提供了多种替代方案：

1. **批处理脚本**：最适合普通用户
2. **Node.js脚本**：最适合开发者
3. **PowerShell脚本**：功能最强大
4. **手动删除**：最直接的方法

这些方法都能有效重置激活码，确保应用的安全性。
